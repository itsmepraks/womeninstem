# Contributing to STEM•SPARK

Thank you for your interest in contributing.

## Code of Conduct

We are committed to providing a welcoming environment for everyone, regardless of gender, gender identity, sexual orientation, disability, race, ethnicity, or age.

### Expected Behavior

- Be respectful and considerate
- Accept constructive criticism
- Focus on what's best for the project
- Show empathy towards others

## How to Contribute

### Reporting Bugs

- Check if the bug has already been reported
- Include steps to reproduce
- Provide screenshots if applicable
- Mention your environment (OS, browser)

### Suggesting Features

- Check if the feature has been suggested
- Provide clear description and rationale
- Include examples if possible

### Contributing Code

#### Setup

1. Fork the repository
2. Clone: `git clone https://github.com/YOUR-USERNAME/womeninstem.git`
3. Create branch: `git checkout -b feature/your-feature`
4. Install dependencies: `npm install`
5. Start dev server: `npm run dev`

#### Code Style

- Use TypeScript for all code
- Follow existing code style
- Run `npm run lint` before committing
- Use meaningful names
- Comment complex logic

#### Commit Messages

Follow Conventional Commits:

```
type(scope): subject
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `style`: Code formatting
- `refactor`: Code refactoring
- `test`: Tests
- `chore`: Maintenance

**Examples:**
```
feat(learning): add progress tracking
fix(ui): resolve navigation issue
docs(readme): update installation steps
```

#### Pull Requests

1. Update documentation if needed
2. Ensure tests pass
3. Update README if needed
4. Request review from maintainers

## Content Guidelines

### Writing Style

- Use clear, direct language
- Include practical examples
- Cite sources

### Content Types

- **Articles**: Educational content
- **Tutorials**: Step-by-step guides
- **Resources**: Tools and references
